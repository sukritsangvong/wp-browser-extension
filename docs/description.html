Project description

Inspired by Controlpedia visualization (Borra et al.), we want to develop an extension that allows users to effortlessly gain insight into the edits made recently. We make an assumption that every good-faith edit made (not including vandalisms) on an article is to “improve” the article in some way. This means that if any portion of the article is being changed, the portion still needs to be “improved” worth notifying the users that what they are reading might not be “reliable” yet.

We want the end product to present users with information about what portions of an article have been recently modified by highlighting portions that went through changes within a given period of time.

We want the users to be able to receive more information on the number of visitors and the number of edits to the article through time series. We also want the users to be able to hover and see some overlay stat on the time series: how many people viewed and edited the article on a specific day.

Users might only be curious about the changes from a certain period of time. Thus, we want the users to be able to pick a time range that will update the version of the article that we will be calculating the differences with the current article with. This can either be done via a drop-down menu or (if possible) by highlighting a certain portion of the time series that the users are interested in.
